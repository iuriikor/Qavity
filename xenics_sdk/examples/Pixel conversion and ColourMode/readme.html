Supported OS: Windows

Description

This sample starts from where we left the "Basic Sample" with different parameters passed to the get_frame method of the XCamera object to specify another type of frame. Also a few new methods are introduced to show how to use different color modes and how to load an image containing a color profile to apply.

The different frame types that can be requested are listed in the table below:

FrameType Description
XFrameType.FT_NATIVE This is the native frame type of the camera.
XFrameType.FT_8_BPP_GRAY Converts the buffer to 8-bit grayscale.
XFrameType.FT_16_BPP_GRAY Converts the buffer to 16-bit grayscale.
XFrameType.FT_32_BPP_GRAY Converts the buffer to 32-bit grayscale.
XFrameType.FT_32_BPP_RGBA Converts the buffer to a 32-bit ABGR format which can hold color information.
XFrameType.FT_32_BPP_RGB Converts the buffer to a 32-bit BGR format which can hold color information.
XFrameType.FT_32_BPP_BGRA Converts the buffer to a 32-bit ARGB format which can hold color information.
XFrameType.FT_32_BPP_BGR Converts the buffer to a 32-bit RGB format which can hold color information.

Used methods

The following methods were used in this sample:

Method Description
XCamera.open Opens a connection to the camera.
XCamera.is_initialized Checks if the camera connection is initialized.
XCamera.start_capture Starts capturing frames from the camera.
XCamera.is_capturing Checks if the camera is capturing.
XCamera.load_colour_profile Loads a color profile image from disk, the dimensions should be 1x256.
XCamera.create_buffer Creates a buffer to store the frame data.
XCamera.get_frame Retrieves a frame from the camera into the buffer.
XCamera.stop_capture Stops capturing frames from the camera.
XCamera.close Closes the connection to the camera.
