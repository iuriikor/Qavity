Supported OS: Windows

Description

This Python script example takes inspiration from the "Basic Sample" script. The primary difference is the frame capturing process. The script captures 5 frames in Non-Blocking mode, followed by 5 frames in Blocking mode, and then 5 frames in NoConversion & Blocking mode.

The blocking mode is determined by the flag parameter, which can take one or multiple values from the XGetFrameFlag-enumeration. These flags can be combined using the bitwise OR-operator.

Flag Description
XGF_Blocking By default, the `get_frame` method will return False when no frame is available. If the programmer wants the function to wait until a frame is available, then this flag is passed.
XGF_NoConversion Internally, for the `get_frame` method, some performance can be gained by setting this flag when grabbing a frame.
XGF_FetchPFF This flag is specific to some types of cameras, like the Onca or Gobi products, and it's not covered in this Python script. It allows to fetch footer information included with the frame coming from the camera, such as timing information, exposure, camera temperature, etc.

Note: The XGF_FetchPFF flag mentioned in the table above is not covered in this Python script sample as it is very specific to the type of camera. For more information about footers, please refer to "Samples -> Python Samples -> Open a camera and get an image -> Footers".
